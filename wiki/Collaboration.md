## 1. Overview of Collaboration Features

Access collaboration tools via **Collaborate** in the main navigation or at **/collaborate**.

| Tool | Description |
|------|------------|
| **Documents** | Real-time collaborative text editor (Google Docs-style) |
| **Wikis** | Structured knowledge base with versioned pages |
| **Tasks** | Todo items with assignees, due dates, and labels |
| **Kanban boards** | Visual task management with drag-and-drop columns |
| **Projects** | Container for multiple boards, documents, and tasks |
| **Meetings** | See [Meetings & Video Calls](Meetings) |

## 2. Documents

### Creating a Document

1. Go to **Collaborate → Documents → + New Document**.
2. Give it a title.
3. Start writing — the editor auto-saves continuously.

### Real-Time Collaboration

Share a document with others for simultaneous editing:
- Multiple users can edit at the same time
- Each collaborator is shown with a distinct cursor colour
- Changes are visible in real time — no refresh needed
- Comments can be added to specific text selections

### Document Editor Features

| Feature | Description |
|---------|------------|
| **Headings** | H1–H6 for structured documents |
| **Tables** | Insert and resize tables |
| **Images** | Embed images from upload or URL |
| **Code blocks** | Syntax-highlighted code |
| **Lists** | Ordered, unordered, and to-do lists |
| **Links** | Insert hyperlinks |
| **Mentions** | @mention collaborators |
| **Comments** | Inline threaded comments |
| **Templates** | Start from a pre-built template |

### Document Visibility

| Option | Who Has Access |
|--------|---------------|
| **Private** | Only you |
| **Shared (by invite)** | Only invited collaborators |
| **Public (view only)** | Anyone with the link can read |
| **Public (editable)** | Anyone with the link can edit |

### Exporting Documents

Export in multiple formats:
1. Click **⋮ → Export**.
2. Choose: **PDF**, **DOCX** (Word), **Markdown**, or **HTML**.

## 3. Wikis

Wikis are structured, multi-page knowledge bases — perfect for documentation, FAQs, and shared reference material.

### Creating a Wiki

1. Go to **Collaborate → Wikis → + New Wiki**.
2. Name the wiki (e.g., "Team Handbook" or "Project Documentation").
3. Add a description.
4. Set visibility.
5. Click **Create**.

### Adding Wiki Pages

1. Inside the wiki, click **+ New Page**.
2. Enter a page title (this becomes the URL slug, e.g., `/wiki/getting-started`).
3. Write content using the rich text editor.
4. Click **Publish**.

### Wiki Navigation

- Pages are listed in the **sidebar** of the wiki.
- Organise pages into **sections and sub-pages** by nesting them in the sidebar.
- Link between wiki pages using `[Page Title](/wiki/page-slug)`.

### Wiki Version History

Every page save creates a version snapshot:
1. Open a page → ⋮ → **Version History**.
2. See the list of saved versions with date, time, and author.
3. Click any version to preview.
4. Click **Restore** to revert to that version.

### Wiki Search

Search within the wiki:
1. Click the 🔍 icon at the top of the wiki sidebar.
2. Type your query.
3. Results highlight matching text within pages.

## 4. Tasks & Issues

Manage to-do items with full task tracking.

### Creating a Task

1. **Collaborate → Tasks → + New Task** (or within a Project board).
2. Fill in:
   - **Title** — what needs to be done
   - **Description** — additional detail (supports markdown)
   - **Assignee(s)** — who is responsible
   - **Due date** — when it should be done
   - **Priority** — Low / Medium / High / Urgent
   - **Labels** — categorise (e.g., Bug, Feature, Design)
   - **Attachments** — files, images
3. Click **Create Task**.

### Task Views

| View | Description |
|------|------------|
| **List** | Flat list of all tasks |
| **Board** | Kanban column view |
| **Calendar** | Tasks on a calendar by due date |
| **Timeline** | Gantt-chart style for project planning |

### Task Statuses

Default statuses (customisable per project):
- **Backlog** — not yet started
- **To Do** — planned for action
- **In Progress** — actively being worked on
- **Review** — awaiting review or approval
- **Done** — completed

## 5. Kanban Boards

Visual project management with drag-and-drop task cards.

### Using a Board

1. **Collaborate → Boards** (or within a Project).
2. Each column represents a stage in your workflow.
3. Drag cards (tasks) between columns to update their status.
4. Click a card to see full details, add comments, or update it.

### Customising Columns

1. Click **+ Add Column** to create a new stage.
2. Drag columns to reorder.
3. Click column title → **Rename** to change its name.
4. Click ⋮ on a column → **Delete Column** (moves cards to backlog first).

### Filtering the Board

Filter visible cards by:
- Assignee
- Priority
- Label
- Due date

## 6. Projects

Projects bring together documents, wikis, boards, and tasks under one roof.

### Creating a Project

1. **Collaborate → Projects → + New Project**.
2. Enter project name and description.
3. Choose a **template** (Software development, Marketing campaign, Event planning, etc.) or start blank.
4. Set visibility.
5. Click **Create Project**.

### Project Components

A project can contain:
- Multiple **Kanban boards**
- A collection of **documents**
- A **wiki** section
- **Task lists**
- **Team members** with roles

### Inviting Team Members

1. Open the project → **⋮ → Members → Invite**.
2. Search for users or share the invite link.
3. Assign a role: **Owner**, **Admin**, **Editor**, **Viewer**.

## 7. Sharing & Permissions

### Document Sharing

1. In any document or wiki, click **Share** (top-right).
2. Choose method:
   - **Invite by username** — send access to a specific user
   - **Share link** — a link anyone can use to access

### Permission Levels

| Level | Can Do |
|-------|--------|
| **Owner** | Full control — edit, delete, manage permissions |
| **Editor** | Read + write (edit content) |
| **Commenter** | Read + leave comments only |
| **Viewer** | Read only |

---

## 8. Version History

All documents and wiki pages maintain full version history.

1. Open the document or page.
2. Click **⋮ → Version History**.
3. See all saved versions with:
   - Author
   - Date and time
   - Change summary (optional note)
4. Click a version to preview.
5. Click **Restore this version** to roll back.

---

## 9. Collaboration in Groups

Groups can have a **Collaboration Space** — a shared area for group documents, wikis, and tasks.

1. In any group, click the **📋 Collaborate** tab.
2. Access shared documents and wikis within that group's space.
3. Only group members can access group collaboration content (based on group privacy).

This is useful for:
- Study groups sharing notes
- Project teams sharing documents
- Community groups building a shared knowledge base
